# Projects sample map

Use this file when the target path is a projects root, research root, or one of its immediate children.

Sample buckets:
- `Active`
- `Shared`
- `Archive`

Project-root rules:
- keep active project folders at the top level
- do not leave project-specific cleanup or archive folders as siblings when the ownership is clear
- move dated exports, repair attempts, or snapshot trees under the owning project's internal `Archive/` folder

Typical active project payload:
- source material
- notes and drafts
- deliverables
- helper scripts
- previews and exports

Grouping rule:
- when loose files at the projects root clearly belong to one project, group them into one folder named by the project stem

Preservation rule:
- preserve already-formed directories such as dated packages, `legacy_*`, or generated-output trees instead of flattening them

This map is a sample, not a required layout.
